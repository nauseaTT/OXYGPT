#
# ⚠️  BUILD NOTE (why the plain `pip install` of the line below fails):
# Telethon v2's build backend generates its TL types at *build* time, and that
# codegen imports `typing_extensions`. pip builds wheels in an ISOLATED
# environment that does NOT include `typing_extensions`, so the build crashes
# with `ModuleNotFoundError: No module named 'typing_extensions'`.
#
# The fix (applied by the Dockerfile and install.sh) is to pre-install the
# build prerequisites and disable build isolation:
#
#     pip install "typing_extensions>=4.12" setuptools wheel
#     pip install --no-build-isolation \
#         "telethon @ git+https://github.com/LonamiWebs/Telethon.git@v2#subdirectory=client"
#
# When Telethon 2.0.0 is published to PyPI, replace this with:  telethon>=2,<3
telethon @ git+https://github.com/LonamiWebs/Telethon.git@v2#subdirectory=client
