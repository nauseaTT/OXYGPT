# Telethon v2 (2.0.0a0). v2 is a full rewrite and is not yet published as a
# stable release on PyPI, so it is installed from the official `v2` branch.
#
# ⚠️  BUILD NOTE (why the plain `pip install` of the line below fails):
# Telethon v2's build backend generates its TL types at *build* time, and that
# codegen imports `typing_extensions`. pip builds wheels in an ISOLATED
# environment that does NOT include `typing_extensions`, so the build crashes
# with `ModuleNotFoundError: No module named 'typing_extensions'`.
#
# The fix (applied by the Dockerfile and install.sh) is to pre-install the
# build prerequisites and disable build isolation:
#
#     pip install "typing_extensions>=4.12" setuptools wheel
#     pip install --no-build-isolation \
#         "telethon @ git+https://github.com/LonamiWebs/Telethon.git@v2#subdirectory=client"
#
# When Telethon 2.0.0 is published to PyPI, replace this with:  telethon>=2,<3
telethon @ git+https://github.com/LonamiWebs/Telethon.git@v2#subdirectory=client

# ── AI providers ──
# The bot uses the NEW unified Google GenAI SDK (`from google import genai`,
# `genai.Client(...)`), which is the `google-genai` package — NOT the legacy
# `google-generativeai` package (`import google.generativeai as genai`).
#
# ⚠️  VERSION PIN (why this is capped): google-genai >= ~1.44 introduced the
# `_interactions` submodule, whose generated TypedDicts use PEP 728's
# `TypedDict(..., extra_items=...)`. That syntax only works on Python 3.13+
# (or a typing_extensions build that supports it at runtime). On this image's
# Python 3.12 base, importing those newer releases fails hard with:
#     TypeError: _TypedDictMeta.__new__() got an unexpected keyword argument 'extra_items'
# 1.10.0 is the last line that imports cleanly on 3.12 while still exposing the
# APIs the bot relies on (genai.Client, google.genai.types, google.genai.errors).
google-genai==1.10.0
openai
